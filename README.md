В рамках данного задания был реализован простой RESTful-сервер на Spring Boot. В качестве доменной области выбран трекер откликов на вакансии, то есть просто хранит отклики на вакансии.

Сервер предназначен для хранения и получения информации об откликах на вакансии. В качестве основной сущности используется JobApplication, которая содержит идентификатор заявки, имя кандидата, название компании, название вакансии, статус отклика и дату создания.

В проекте реализованы 3 endpoint’а:

GET /applications — получение списка всех откликов;
POST /applications — создание нового отклика по JSON-телу запроса;
GET /applications/{id} — получение отклика по его идентификатору.

Для создания новой заявки используется отдельный DTO CreateJobApplicationDTO. Для входных данных добавлена базовая валидация через @Valid и @NotBlank, чтобы не допускать пустые обязательные поля.

Хранение данных реализовано in-memory в сервисном слое через LinkedHashMap, а идентификаторы заявок генерируются последовательно с помощью AtomicLong.

Также в проект был добавлен RequestLoggingFilter, который логирует входящие HTTP-запросы, включая метод, URI, content type и тело запроса. Для корректного чтения тела используется ContentCachingRequestWrapper.
